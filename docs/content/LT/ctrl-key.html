^H はなぜ BS なのか

bash 等で ^H (Ctrl + h) を押すと BS のように振る舞う．
なぜだろうか？

A. Ctrl キーは元来，制御文字を送るためのキーだから (?)

テレタイプ端末、及び初期のコンピュータキーボードでは、コントロールキーを押しながら他のキーを押すと、生成されるASCIIの下位5ビット以外がゼロとなった。 これによりユーザーはASCIIの非表示文字である最初の32文字 (0x00 - 0x1f) を生成、入力できる。 (Wikipedia)

ASCII 表を見ると確かに色々と対応している．

下位5bit\上位3bit * 000
00100 d EOT
01000 h BS
01001 i HT
01010 j LF
01101 m CR
11011 [ ESC
